Posted on July 22nd, 2016

Over the past few days I've done a lot of thinking and some engineering to change how this website is to be run. I've been picking up Ruby on Rails because HTML/CSS is the best way to display a static website, but has next to no functionalities when it comes to interactive capabilities. And while I don't see the need for a dynamic interaction for this website per se, I do think that by learning different capabilities I would be able to better display what/how I want, without technical capabilities.

The first step has been reorganizing the Thoughts page, which used to be a long stream of text on a single page. Sure, it's not a problem as of now, but as the number of posts increase, I felt that there would be a need to better organize posts. An ideal way to do this would be to implement a calendar functionality as well as a categorization of my thoughts for better organization, but honestly I have no idea how to implement tags, let alone calendars, at this point (now that I think of it, I do think that HTML offers some type of calendar, I should look into it).

The second part has been Bootstrap, which I've started to pick up for aesthetics' sake. The index page has already been redesigned with Bootstrap, but I've been stuck on doing the same for the Thoughts page, especially with regards to implementing Bootstrap and my own stylesheet at the same time.

At the end of the day, the skills I pick up and the practice I do with this website(which is effectively acting as my own sandbox) will help me engineer the Cloud IR project, as well as the Giles 2016 page. The idea now for Giles 2016 is that I would have students ask questions on the site, and I would publicly display the ones that I think would benefit other students as well, while filtering out the useless ones(perhaps I could answer them without posting them?). As long as I figure out how to actually do it, the process of making the sites sounds quite intriguing, to say the least.

Posted on July 3rd, 2016

So, for the last three weeks I've been creating the anniversary project website for Nicole. The idea first hit me out of nowhere - quite literally, in that I was biking home after work, and the idea just struck me. I guess a lot of it had to do with the fact that I was just picking up basic HTML/CSS and JavaScript tutorials from Khan Academy. While racking my brains for what to get her, especially since she didn't want flowers(my go-to present up until now), I came up with an idea: why don't I create something from scratch? That way, it'll be a truly one-of-a-kind gift, one that I'd poured my heart into.

As I soon discovered, things were easier said than done. I had a whole idea sketched on paper, but implementing the nitty details via HTML/CSS wasn't the easiest thing in the world, at least partially because I had just started teaching myself the details. Minute implementations often took hours of Googling plus trial and error, and honestly, if it weren't for the fact that I had to make this work, I would probably have met myself halfway between and compromised with what I could make.

What started out as a small idea developed into a full-scale idea that would take weeks to complete. Not only did I have to create the skeletal structure for the website, I would also fill each page with a lengthy letter, complete with pictures. I've never been happier with iPhotos, which not only copies every photo from my iPhone to my MacBook, but also enables easy file transfers so that I wouldn't have to use a cloud server or anything. Best of all, going through all the pictures of us throughout the past year really made me realize just how much Nicole means to me.

The webpage has been hosted as a Github Project Page, with an external domain purchased through Google domain. Everything was a first for me, and it took some errors to get everything correct, but now things are good to go.

The link is as follows: raymeetsnicole.com. I plan on continuously updating the website with more of our adventures, so hold on tight - it's going to be a bumpy ride!
